Automation in Practice

A real retention system I designed end-to-end using Airtable and Make.com.

Student Retention Dashboard & Automation

Built a complete retention workflow for tracking student course progress. Airtable holds the live student records; a scheduled Make.com automation scans those records every minute, branches by status (at-risk, inactive, active, completed), and sends the right follow-up email through Gmail — then logs the outcome back to the record. The same data powers a live reporting dashboard.

  • Designed a multi-branch Make.com scenario that routes each student down a different automated email path based on their status
  • Combined HTTP, text parser, Gmail, and Airtable modules into one reliable, scheduled workflow
  • Structured the Airtable base — status, percent complete, engagement trend, testimonial replies, last activity — to serve both the automation and the dashboard
  • Built and deployed a lightweight, real-time web dashboard reading from the same base
